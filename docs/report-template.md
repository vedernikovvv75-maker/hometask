# Отчёт по практике — Lead Intake MVP

## 1) Кому и какую задачу решает проект
Клиент: небольшой бизнес, где лиды приходят из разных источников и теряются.  
Проект автоматизирует путь заявки: приём через webhook, сохранение в SQLite и фиксация события о принятии.

## 2) Входные данные
Endpoint: `POST /lead`  
JSON-формат:

```json
{
  "name": "Ирина",
  "contact": "+79990000000",
  "source": "landing",
  "comment": "Хочу консультацию по тарифам"
}
```

Обязательное поле: `contact` (также валидируются все поля из схемы).

## 3) Выход / результат
- Лид сохраняется в SQLite (`data/leads.db`, таблица `leads`).
- Сервис пишет событие в `logs/events.log` в формате `New lead saved: <id>`.

## 4) Архитектура
`POST /lead` -> `validation` -> `sqlite insert` -> `event log`.
